# **FSJES UCA Results+ - Complete Application Description**

## **1. Application Overview**

**Application Name:** FSJES UCA Results+  
**Tagline:** "Your Academic Results, Smarter and More Beautiful!"  
**Platform:** iOS & Android  
**Target Users:** Students of Faculty of Legal, Economic and Social Sciences - Cadi Ayyad University (FSJES UCA)

**Core Concept:**  
A modern, feature-rich mobile application that automatically retrieves and displays academic results from the official FSJES UCA portal. The app transforms raw data into an intuitive, visually appealing interface with customization options and smart features tailored for students.

---

## **2. Key Features & Functionality**

### **2.1 Modern User Interface & Experience**
- **Material Design 3 Implementation:** Clean, modern interface with consistent design language
- **Dual Theme System:** 
  - Light mode (default)
  - Dark mode (reduces eye strain, saves battery)
  - Seamless switching via settings
- **Smooth Animations:** 
  - Loading animations during data retrieval
  - Transition animations between screens
  - Micro-interactions for button presses
- **Responsive Layout:** Adapts to all screen sizes and orientations

### **2.2 Smart Results Display**
- **Comprehensive Subject Information:**
  - Subject name clearly displayed alongside grade
  - Grade value with appropriate precision
  - Status indicator with visual cues
- **Color-Coded Results:**
  - **Green:** Passed subjects (V, AC)
  - **Red:** Failed or retake required (NV)
  - **Orange/Gray:** Special cases (ABJ, ABI)
- **Academic Summary:**
  - Semester GPA calculation
  - Cumulative GPA tracking
  - Credits earned vs. total
- **Eliminated Issues:**
  - No duplicate grade display
  - No unnecessary true/false options
  - Clear subject-grade pairing

### **2.3 Multi-Language Support**
- **Three Languages:** Arabic, French, English
- **Dynamic Switching:** Change language anytime without restarting app
- **Complete Translation:**
  - All interface elements
  - Status descriptions
  - Help text and instructions
- **Language Persistence:** Remembers user preference

### **2.4 Data Source & Processing**
- **Automated Data Retrieval:**
  - Connects to: `https://e-apps.fsjes.uca.ma/scolarite/resultat/index.php?apogee=[STUDENT_ID]`
  - Secure student ID input with validation
- **Smart Parsing Engine:**
  - Extracts: Subject names, grades, status codes
  - Converts codes to readable formats:
    - V → Passed
    - NV → Failed/Retake Required
    - AC → Passed through Integration
    - ABJ → Justified Absence
    - ABI → Unjustified Absence
- **Data Validation:** Checks for format consistency and completeness

### **2.5 Customization & Settings**
- **Theme Control:**
  - Light/Dark mode toggle
  - Accent color selection
  - Font size adjustment
- **Display Preferences:**
  - Grade rounding options
  - GPA scale selection (4.0, 20-point, etc.)
  - Notification preferences
- **Privacy Settings:**
  - Clear local data option
  - Export results as PDF/Excel

### **2.6 Offline Functionality**
- **Local Storage:** Saves results for offline access
- **Cache Management:** Intelligent data updating
- **Export Capabilities:** Share results via email/messaging apps

## **4. Technical Architecture**

### **4.1 Development Framework**
- **Primary Choice:** Flutter (Dart)
  - Single codebase for iOS & Android
  - Rich widget library
  - Excellent performance
- **Alternative:** React Native (JavaScript/TypeScript)

### **4.2 Core Libraries & Dependencies**
- **HTTP Client:** Dio or http for API calls
- **HTML Parsing:** html or flutter_html for web scraping
- **Local Storage:** Hive or SharedPreferences
- **State Management:** Provider or Riverpod
- **Internationalization:** flutter_localizations
- **PDF Generation:** pdf or printing
- **Charts:** fl_chart or charts_flutter

### **4.3 Data Flow**
```
User Input → ID Validation → Web Request → HTML Parsing → 
Data Processing → Local Storage → UI Update → User Display
```

### **4.4 Security Considerations**
- No storage of passwords
- Local data encryption
- Secure HTTP requests
- Privacy disclaimer
